# События Февраля

Планируемые мероприятия и задачи на февраль в рамках курса "Программная инженерия".

## Обзор месяца

Февраль - месяц углубленного изучения DevOps практик и управления проектами. Основной фокус на автоматизацию процессов и командную работу.

## Недели февраля

### Неделя 1 (1-4 февраля)

**Тема**: Расширенные возможности GitHub Actions

**Мероприятия**:
- 🎓 **Лекция**: Advanced GitHub Actions
  - Дата: 1 февраля, 10:00
  - Темы: Matrix builds, artifacts, caching, environments
  
- 💻 **Практикум**: Создание сложных workflows
  - Дата: 3 февраля, 14:00
  - Практика работы с зависимостями и артефактами

**Задачи**:
- [ ] Настроить matrix build для нескольких платформ
- [ ] Реализовать кэширование зависимостей
- [ ] Настроить публикацию артефактов сборки

**Дедлайны**:
- Расширенный workflow: до 4 февраля

**Задачи**:
- [ ] Создать Project Board для команды
- [ ] Распланировать первый спринт
- [ ] Настроить автоматизацию в Projects
- [ ] Создать roadmap проекта

**Дедлайны**:
- Настройка Project Board: до 11 февраля

## Проектная работа

В феврале начинается работа над учебным проектом:

### Этапы проекта
1. **Планирование** (5-11 февраля)
   - Определение требований
   - Создание технического задания
   - Распределение задач

2. **Разработка** (12-25 февраля)
   - Реализация функциональности
   - Написание тестов
   - Code reviews

3. **Релиз** (26-28 февраля)
   - Подготовка к релизу
   - Развертывание
   - Презентация результатов

## Оценка и критерии

### Критерии оценки за февраль
- **Технические задачи** (40%): выполнение всех практических заданий
- **Проектная работа** (30%): вклад в командный проект
- **Качество кода** (20%): code reviews, тесты, документация
- **Активность** (10%): участие в обсуждениях, помощь коллегам

### Требования для зачёта
- ✅ Выполнены все обязательные задачи
- ✅ Пройдены все дедлайны (или с минимальной задержкой)
- ✅ Активное участие в code reviews
- ✅ Вклад в проектную работу
