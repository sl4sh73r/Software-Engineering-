# Программная инженерия (Software Engineering)

## Описание проекта

Этот проект создан для курса "Программная инженерия" и используется в качестве альтернативы Azure DevOps с использованием GitHub. Репозиторий предназначен не только для документации проекта, но и для практических занятий по DevOps, включая:

- Настройку CI/CD пайплайнов
- Автоматические сборки
- Управление конфигурациями
- Практики непрерывной интеграции и доставки

## Документация

Подробная документация проекта доступна в [Wiki](../../wiki) этого репозитория.

## Структура проекта

```
Software-Engineering-/
├── .github/
│   └── workflows/        # GitHub Actions CI/CD конфигурации
├── docs/                 # Документация проекта
├── README.md            # Этот файл
├── CONTRIBUTING.md      # Руководство по внесению изменений
└── Wiki/                # Исходные файлы Wiki (для локальной работы)
```

- **`.github/workflows/`** - содержит файлы конфигурации для GitHub Actions (CI/CD пайплайны)
- **`docs/`** - дополнительная документация проекта
- **`Wiki/`** - локальные файлы для Wiki репозитория
- **`CONTRIBUTING.md`** - правила и рекомендации по работе с репозиторием

## Как работать с документацией

1. **Просмотр документации**: основная документация находится в [Wiki](../../wiki) репозитория
2. **Редактирование через веб-интерфейс**: перейдите во вкладку Wiki и используйте кнопку "Edit"
3. **Локальное редактирование**: клонируйте Wiki репозиторий отдельно (см. CONTRIBUTING.md)
4. **Добавление новых страниц**: создайте новый Markdown файл в Wiki

Подробные инструкции по работе с документацией описаны в файле [CONTRIBUTING.md](CONTRIBUTING.md).

## Что будет дальше

В рамках курса "Программная инженерия" мы будем последовательно реализовывать следующие практики:

### Планирование и управление проектом
- Создание и управление задачами (Issues)
- Использование Projects для канбан-досок
- Управление календарём поставок и релизами
- Планирование спринтов и итераций

### DevOps практики
- **Настройка CI/CD**: автоматизация сборки, тестирования и развертывания
- **GitHub Actions**: создание и настройка пайплайнов в формате YAML
- **Автоматические сборки**: конфигурация триггеров и условий запуска
- **Тестирование**: интеграция автоматических тестов в пайплайн
- **Развертывание**: автоматическая доставка приложений

### Управление конфигурациями
- Работа с переменными окружения
- Управление секретами
- Конфигурационные файлы для различных сред

## Начало работы

Для начала работы с проектом:

1. Ознакомьтесь с [Wiki](../../wiki) проекта
2. Прочитайте [CONTRIBUTING.md](CONTRIBUTING.md) для понимания процесса работы
3. Изучите существующие Issues и Projects
4. Настройте локальное окружение для разработки (см. Wiki → Developer Environment)
