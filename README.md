## 🚀 Шаги практики

### Шаг 1: Подготовка Wiki/Docs

**Цель**: Создать место для хранения документации проекта.

#### Вариант А: Использование папки `docs`

1. Создайте папку `docs` в корне репозитория:
   ```bash
   mkdir docs
   ```

2. Создайте главную страницу `Домашняя.md`:
   ```bash
   cd docs
   touch Домашняя.md
   ```

3. Добавьте базовое содержимое в `Домашняя.md`:
   ```markdown
   # Домашняя страница
   
   Добро пожаловать в документацию проекта Software Engineering!
   
   ## Содержание
   - [Начало работы](Начало-работы.md)
   - [Окружение для разработки](Окружение-для-разработки.md)
   ```

#### Вариант Б: Использование GitHub Wiki

1. Перейдите в свой репозиторий на GitHub
2. Нажмите на вкладку **Wiki** (находится в верхнем меню)
3. Нажмите **Create the first page**
4. Создайте страницу "Домашняя" (Home)

💡 **Совет**: Папка `docs` лучше подходит для версионирования вместе с кодом, а Wiki — для быстрого редактирования через веб-интерфейс.

### Шаг 2: Добавление страниц документации

**Цель**: Создать базовые страницы с информацией о проекте.

#### Страница "Начало работы"

1. Создайте файл `docs/Начало-работы.md`:
   ```bash
   touch docs/Начало-работы.md
   ```

2. Добавьте содержимое:
   ```markdown
   # Начало работы
   
   ## Введение
   Этот раздел поможет вам быстро начать работу с проектом.
   
   ## Первые шаги
   1. Склонируйте репозиторий
   2. Установите зависимости
   3. Запустите проект
   
   ## Дополнительные ресурсы
   - [Документация GitHub](https://docs.github.com)
   - [Markdown Guide](https://www.markdownguide.org)
   ```

#### Страница "Окружение для разработки системы"

1. Создайте файл `docs/Окружение-для-разработки.md`:
   ```bash
   touch docs/Окружение-для-разработки.md
   ```

2. Добавьте системные требования:
   ```markdown
   # Окружение для разработки системы
   
   ## Системные требования
   
   ### Минимальные требования:
   - **ОС**: Windows 10/11, macOS 10.15+, Linux (Ubuntu 20.04+)
   - **Процессор**: Intel Core i3 или аналог
   - **ОЗУ**: 4 ГБ
   - **Дисковое пространство**: 10 ГБ
   
   ### Рекомендуемые требования:
   - **ОС**: Windows 11, macOS 12+, Linux (Ubuntu 22.04+)
   - **Процессор**: Intel Core i5 или аналог
   - **ОЗУ**: 8 ГБ
   - **Дисковое пространство**: 20 ГБ
   
   ## Необходимое ПО
   - Git 2.30+
   - Текстовый редактор (VS Code, Sublime Text, Atom)
   - Node.js 16+ (опционально)
   - Python 3.8+ (опционально)
   
   ## Настройка окружения
   1. Установите Git
   2. Настройте SSH-ключи для GitHub
   3. Установите редактор кода
   4. Склонируйте репозиторий
   ```

### Шаг 3: Внесение изменений и коммиты

**Цель**: Зафиксировать изменения в системе контроля версий.

1. Добавьте все созданные файлы в индекс Git:
   ```bash
   git add docs/
   ```

2. Создайте коммит с описанием изменений:
   ```bash
   git commit -m "Добавлены страницы Домашняя, Начало работы и Окружение для разработки"
   ```

3. Отправьте изменения на GitHub:
   ```bash
   git push origin main
   ```

4. Внесите изменения в существующие страницы (например, добавьте эмодзи):
   ```bash
   # Отредактируйте файл docs/Домашняя.md
   git add docs/Домашняя.md
   git commit -m "Обновлена главная страница документации"
   git push origin main
   ```

💡 **Совет**: Делайте коммиты часто и с понятными сообщениями. Это поможет отслеживать историю изменений.

### Шаг 4: Создание структуры для событий

**Цель**: Организовать документацию в иерархическую структуру.

1. Создайте папку для событий:
   ```bash
   mkdir -p docs/Предстоящие-события
   ```

2. Создайте файл `События-Января.md`:
   ```bash
   touch docs/Предстоящие-события/События-Января.md
   ```
   
   Содержимое:
   ```markdown
   # События Января
   
   ## 📅 Календарь событий на январь
   
   ### Неделя 1 (1-7 января)
   - **5 января**: Встреча команды по планированию спринта
   - **7 января**: Ревью кода
   
   ### Неделя 2 (8-14 января)
   - **10 января**: Демонстрация функционала заказчику
   - **12 января**: Технический вебинар
   
   ### Неделя 3 (15-21 января)
   - **17 января**: Командный хакатон
   - **19 января**: Обучающая сессия по Git
   
   ### Неделя 4 (22-31 января)
   - **24 января**: Релиз версии 1.0
   - **26 января**: Ретроспектива спринта
   ```

3. Создайте файл `События-Февраля.md`:
   ```bash
   touch docs/Предстоящие-события/События-Февраля.md
   ```
   
   Содержимое:
   ```markdown
   # События Февраля
   
   ## 📅 Календарь событий на февраль
   
   ### Неделя 1 (1-7 февраля)
   - **2 февраля**: Планирование нового спринта
   - **4 февраля**: Code review сессия
   
   ### Неделя 2 (8-14 февраля)
   - **9 февраля**: Презентация новых фич
   - **11 февраля**: Встреча с архитектором
   
   ### Неделя 3 (15-21 февраля)
   - **16 февраля**: Командный митинг
   - **18 февраля**: Техническая встреча
   
   ### Неделя 4 (22-28 февраля)
   - **23 февраля**: Выходной день
   - **25 февраля**: Подведение итогов месяца
   ```

4. Зафиксируйте изменения:
   ```bash
   git add docs/Предстоящие-события/
   git commit -m "Добавлена структура событий января и февраля"
   git push origin main
   ```

### Шаг 5: Настройка навигации

**Цель**: Упростить навигацию по документации с помощью боковой панели.

1. Создайте файл `_Sidebar.md` в папке `docs`:
   ```bash
   touch docs/_Sidebar.md
   ```

2. Добавьте структуру навигации:
   ```markdown
   # 📚 Навигация
   
   ## Основные разделы
   - [🏠 Домашняя](Домашняя.md)
   - [🚀 Начало работы](Начало-работы.md)
   - [⚙️ Окружение для разработки](Окружение-для-разработки.md)
   
   ## 📅 События
   - [Предстоящие события](Предстоящие-события/)
     - [События Января](Предстоящие-события/События-Января.md)
     - [События Февраля](Предстоящие-события/События-Февраля.md)

   **[⬆️ Наверх](#-навигация)**
   ```

3. Также можно создать файл `index.md` в папке `docs` как индексный файл:
   ```bash
   touch docs/index.md
   ```
   
   Содержимое:
   ```markdown
   # Индекс документации
   
   ## Добро пожаловать!
   
   Это централизованный индекс всей документации проекта.
   
   ### Быстрые ссылки
   - [Домашняя страница](Домашняя.md)
   - [Начало работы](Начало-работы.md)
   - [Окружение для разработки](Окружение-для-разработки.md)
   
   ### События
   - [События Января](Предстоящие-события/События-Января.md)
   - [События Февраля](Предстоящие-события/События-Февраля.md)
   ```

4. Сохраните изменения:
   ```bash
   git add docs/_Sidebar.md docs/index.md
   git commit -m "Добавлена боковая панель навигации и индексный файл"
   git push origin main
   ```

💡 **Совет**: Файл `_Sidebar.md` автоматически распознаётся GitHub Wiki как боковая панель.

### Шаг 6: Проверка поиска и навигации

**Цель**: Убедиться, что документация работает правильно.

#### В папке `docs`:

1. **Проверка структуры**:
   ```bash
   tree docs/
   # или
   ls -R docs/
   ```

2. **Проверка ссылок**: Откройте файлы в VS Code или другом редакторе и убедитесь, что все ссылки корректны.

3. **Локальный просмотр**: Используйте расширения для VS Code (например, Markdown Preview) или установите локальный сервер:
   ```bash
   # С помощью Python
   cd docs
   python3 -m http.server 8000
   # Откройте http://localhost:8000 в браузере
   ```

#### В GitHub Wiki:

1. Перейдите на вкладку **Wiki** в вашем репозитории
2. Проверьте, что все страницы отображаются корректно
3. Используйте поиск (иконка лупы) для проверки поисковой функции
4. Проверьте навигацию по ссылкам между страницами
5. Проверьте историю изменений (кнопка **Page History**)

## 📖 Как работать с документацией

### Где писать страницы?

- **В папке `docs/`**: Создавайте файлы с расширением `.md` (Markdown)
- **В GitHub Wiki**: Используйте веб-интерфейс GitHub для создания страниц

### Как добавлять новые страницы?

#### В папке `docs/`:
```bash
# Создайте новый файл
touch docs/Новая-Страница.md

# Добавьте содержимое (используйте редактор)
nano docs/Новая-Страница.md

# Сохраните в Git
git add docs/Новая-Страница.md
git commit -m "Добавлена новая страница документации"
git push origin main
```

#### В GitHub Wiki:
1. Откройте вкладку **Wiki**
2. Нажмите **New Page**
3. Введите название и содержимое
4. Нажмите **Save Page**

### Как править существующие страницы?

#### В папке `docs/`:
```bash
# Отредактируйте файл в редакторе
nano docs/Домашняя.md

# Сохраните изменения
git add docs/Домашняя.md
git commit -m "Обновлена домашняя страница"
git push origin main
```

#### В GitHub Wiki:
1. Откройте нужную страницу
2. Нажмите **Edit**
3. Внесите изменения
4. Нажмите **Save Page**

### Как смотреть историю изменений?

#### В папке `docs/`:
```bash
# Показать историю коммитов для файла
git log docs/Домашняя.md

# Показать изменения в файле
git log -p docs/Домашняя.md

# Показать, кто и когда менял каждую строку
git blame docs/Домашняя.md
```

#### В GitHub Wiki:
1. Откройте нужную страницу
2. Нажмите **Page History** (или **Revisions**)
3. Просмотрите список изменений
4. Нажмите на коммит для просмотра деталей

#### Через веб-интерфейс GitHub:
1. Перейдите в папку `docs/` в репозитории
2. Откройте нужный файл
3. Нажмите **History** (справа сверху)
4. Просмотрите историю изменений

## 🛠️ Полезные команды Git и GitHub

### Базовые команды Git

```bash
# Клонирование репозитория
git clone https://github.com/username/repository.git

# Проверка статуса изменений
git status

# Добавление файлов в индекс
git add <file>              # Добавить конкретный файл
git add .                   # Добавить все изменения
git add docs/               # Добавить всю папку

# Создание коммита
git commit -m "Описание изменений"

# Отправка изменений на GitHub
git push origin main

# Получение изменений с GitHub
git pull origin main

# Просмотр истории коммитов
git log
git log --oneline           # Краткий вид
git log --graph --all       # С графом веток

# Просмотр изменений
git diff                    # Изменения в рабочей директории
git diff --cached           # Изменения в индексе
```

### Работа с ветками

```bash
# Создание новой ветки
git branch feature-docs

# Переключение на ветку
git checkout feature-docs

# Создание и переключение одной командой
git checkout -b feature-docs

# Слияние ветки
git checkout main
git merge feature-docs

# Удаление ветки
git branch -d feature-docs
```

### Полезные команды для документации

```bash
# Проверка структуры папок
tree docs/                  # Требует установки tree
ls -R docs/                 # Рекурсивный список

# Поиск в файлах документации
grep -r "искомый текст" docs/

# Подсчет строк в документации
wc -l docs/*.md

# Проверка Markdown (требует markdownlint)
markdownlint docs/
```

### GitHub CLI команды

Если у вас установлен [GitHub CLI](https://cli.github.com/):

```bash
# Создание репозитория
gh repo create

# Просмотр pull requests
gh pr list

# Создание issue
gh issue create

# Просмотр веток
gh repo view

# Клонирование репозитория
gh repo clone username/repository
```

### Дополнительные советы

- 📝 **Используйте осмысленные сообщения коммитов**: Описывайте, что и зачем изменено
- 🔄 **Делайте коммиты часто**: Маленькие коммиты легче понять и откатить
- 📚 **Читайте документацию**: [GitHub Docs](https://docs.github.com), [Git Book](https://git-scm.com/book)
- 🎯 **Используйте .gitignore**: Исключайте временные и служебные файлы
- 🔍 **Проверяйте изменения перед коммитом**: Используйте `git diff`
